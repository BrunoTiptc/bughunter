📌 Requisito Funcional – RF001 – Login de Usuário
📖 Descrição

O sistema deve permitir que o usuário insira email e senha para acessar a aplicação.

🎯 Objetivo

Permitir autenticação de usuário na plataforma BugHunter.

🧩 Entradas

Campo email (obrigatório)

Campo senha (obrigatório)

Botão "Entrar"

✅ Regras de Negócio

Ambos os campos são obrigatórios.

O sistema deve exibir mensagem de erro se algum campo estiver vazio.

O sistema deve permitir envio do formulário ao clicar no botão.

(Futuro) Backend deve validar credenciais.

🔎 Critérios de Aceite

 O campo email aceita texto.

 O campo senha mascara os caracteres.

 O botão só funciona se os campos estiverem preenchidos.

 O sistema exibe mensagem se campos vazios.
